# AI Panel Studio 工作流说明

## 开发流程

本项目按照 SDD + DDD + TDD 的方式拆解开发。

1. SDD 阶段先定义业务对象、API 契约、Prompt 输出格式和安全约束。
2. DDD 阶段将圆桌演播厅拆分为可维护的 UI 组件与状态区域。
3. TDD 阶段先覆盖讨论引擎、Prompt 规则和 API 请求参数。
4. 后续 E2E 阶段会验证完整用户路径。

## 典型问题

- 问题一: AI 输出容易混入 Markdown 或解释文本。
  解决: Prompt 中明确要求 JSON，并在后端集中解析。

- 问题二: 专家轮流发言会显得机械。
  解决: 调度 Prompt 要求基于 transcript 自主选择下一位发言者。

- 问题三: 内部事件不应显示给用户。
  解决: Transcript 类型只保存用户可见发言，内部状态单独维护。
